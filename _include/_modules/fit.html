orsa.fit¶

This submodule can be used to fit spectra.

class orsa.fit.LineProfiler¶

A profiler that records the execution times of individual lines.

This provides the core line-profiler functionality.

Example

>>> import line_profiler
>>> profile = line_profiler.LineProfiler()
>>> @profile
>>> def func():
>>>     x1 = list(range(10))
>>>     x2 = list(range(100))
>>>     x3 = list(range(1000))
>>> func()
>>> profile.print_stats()

orsa.fit.emcee(f: type[CostFunction], n_samples: int, err_scale: float = 0.001, err_scale_is_relative: bool = False, backend: str = None, reset_backend: bool = True, ll_scale: float = None, lp_scale: float = 1.0, parallel: bool = False, emcee_kwargs: dict = {}, plot_progress: bool = False, best_values: str = 'median') → type[Results]¶

Function to perform a fit through the emcee library.

Parameters:

  • f (CostFunction) – cost function to use for the minimization.

  • n_samples (int) – number of values to sample for each walker. You will obtain (n_samples x 2 x n_parameters) samples at the end of the MCMC.

  • err_scale (float, optional) – width of the bubble (in absolute units by default) to use for the inizialization of the workers around the initial value. Defaults to 1e-3.

  • err_scale_is_relative (bool, optional) – specify whether the err_scale is provided in absolute or relative units. Defaults to False (i.e. errors are in absolute scale).

  • backend (str, optional) – path to the file in which emcee stores the chains. Defaults to None.

  • reset_backend (bool, optional) – if backend file with the same name already exists, decide wether to reset it before the fit or continue sampling from the latest saved point. Defaults to True.

  • ll_scale (float, optional) – possible value to override the default ll_scale parsed by the function name. Defaults to None.

  • lp_scale (float, optional) – possible value to override the default lp_scale for optimizers. Defaults to 1..

  • parallel (bool, optional) – flag to enable the parallelization of the chains in different threads. Defaults to False.

  • emcee_kwargs (dict, optional) – dictionary of kwargs to pass to the emcee sampler. Defaults to {}.

  • plot_progress (bool, optional) – WIP flag to enable the real-time plotting of the MCMC chains. Defaults to False.

  • best_values (str, optional) – WIP method for the estimations of best fit values from the chains. Defaults to ‘median’.

Returns:

result of the fit.

Return type:

Result

orsa.fit.lru_cache(maxsize=128, typed=False)¶

Least-recently-used cache decorator.

If maxsize is set to None, the LRU features are disabled and the cache can grow without bound.

If typed is True, arguments of different types will be cached separately. For example, f(3.0) and f(3) will be treated as distinct calls with distinct results.

Arguments to the cached function must be hashable.

View the cache statistics named tuple (hits, misses, maxsize, currsize) with f.cache_info(). Clear the cache and statistics with f.cache_clear(). Access the underlying function with f.__wrapped__.

See: https://en.wikipedia.org/wiki/Cache_replacement_policies#Least_recently_used_(LRU)
